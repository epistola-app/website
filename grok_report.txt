**1. How can we convince the user to use Epistola**

Your current hero and "Why Epistola" sections already do a strong job of positioning it as the **single-purpose, trustworthy engine** that "turns structured data into perfect PDFs" without AI bloat or workflow drag. That's a powerful differentiator in a market full of over-engineered tools.

To make conviction stronger and faster (like Stripe or Supabase do instantly):

- **Lead with the pain + promise in the hero**: Change the subheadline to something benefit-first like "Stop fighting PDF fidelity in regulated workflows. Model once, render perfectly every time — self-hosted, open-source, and built for orchestrators like Valtimo/ZGW." Add a one-click "Deploy with Docker" or "See live editor demo" CTA right under it.
- **Add immediate social proof**: Replace placeholder logos and anonymous "Customer Name" testimonials with real ones (names, photos, companies, and quantifiable wins like "Cut document errors from 12% to 0% and saved 3 weeks per release"). Stripe and Carbone.io nail this with named enterprise stories (Hertz, Salesforce users).
- **Show, don't just tell**: The biggest missing piece is **trust through demonstration**. "Introduction video coming soon" is a huge conversion killer. Replace it with a 60-second loom-style video or embedded GIF of the visual editor + live PDF preview. Users in regulated industries buy when they see compliance in action (PDF/A, version gating, audit trails).
- **Offer a zero-friction next step**: Add a prominent "Try the editor in browser" sandbox (even read-only) or "One-click self-host demo on GitHub Codespaces/Docker Play". Supabase and PDFMonkey convert visitors by letting them play immediately.

This turns your "we're the reliable one" message into "try it and you'll never go back."

**3. What can we do differently compared to other similar landing/documentation websites of very good companies**

Top performers (Stripe, Supabase, Vercel, Carbone.io, PDFMonkey) all share these traits that Epistola currently lacks:

- **Instant interactivity** — Stripe has live checkout demos, PDFMonkey has real-time template preview in the dashboard, Supabase has copy-paste starter templates. You have none.
- **Clear quickstart path** — Every great site has "Get started in 5 minutes" with code snippet + deploy button. You have excellent "How it works" steps but no copy-paste OpenAPI example or Docker command visible above the fold.
- **Hybrid landing + docs** — Supabase/Vercel have seamless nav between marketing and full docs with search. You’re currently pure landing with no searchable docs sidebar.
- **Developer-first details** — Vercel and Stripe show code snippets everywhere. Add a live "Render this JSON → PDF" playground on the homepage.
- **Differentiation you already have (lean into it)**: Most competitors (Carbone, PDFMonkey) are either closed-source or HTML-to-PDF (which breaks fidelity). You’re direct PDF renderer + native Common Ground + multi-tenant governance + open-source self-host. That’s your killer combo — make it the hero differentiator instead of burying it in "Why Epistola".
